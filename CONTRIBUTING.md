# Contributing to Rubicon Docker Best Practices

Thanks for your interest in contributing! Here are some ways you can help:

## Reporting Bugs

Please open an issue with:

- Clear title
- Description of the problem
- Steps to reproduce
- Environment info (Synology version, container, etc.)

## Improvements

Submit a PR if you want to:

- Add new container examples (e.g., Plex, Home Assistant, etc.)
- Improve docs or fix typos
- Add new labels, security tips, or CI enhancements

## Pull Request Guidelines

- Keep changes atomic and focused
- Use clear commit messages
- Confirm your docker-compose.yaml passes `docker compose config`

Thanks again!
